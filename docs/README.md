# Design docs

Planning and design documents for this Pi package. These describe intent and
roadmap; they are not auto-loaded by Pi.

- [`research-workflow-improvement-plan.md`](./research-workflow-improvement-plan.md)
  — The master roadmap for evolving this package from "custom tools + providers"
  into a research operating system for Starlit Labs systems work (Axiom /
  Hawkeye / Saga / eBPF), with PhD kept optional. Defines the phased plan
  (Phase 0 housekeeping → web ingestion → paper tools → capture tools →
  research session → skills/prompts → compaction → subagents), tool schema
  sketches, and the eight design principles (artifact-first, citation-preserving,
  chunkable, no-surprise-autonomy, research-now/PhD-optional, Starlit Labs only,
  small composable tools, surgical hook use).

When adding a new design doc, add an entry here and keep the planning doc as the
authoritative sequence source.
